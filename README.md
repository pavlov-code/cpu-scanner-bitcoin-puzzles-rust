# cpu-scanner-bitcoin-puzzles-rust
Инструмент для поиска приватных ключей Bitcoin по RIPEMD-160 хешам с графическим интерфейсом и автоматической «каруселью» диапазонов для BitCoin Puzzles

🔐 **Bitcoin365 Carousel Searcher**  
*Инструмент для поиска приватных ключей Bitcoin по RIPEMD-160 хешам с графическим интерфейсом и автоматической «каруселью» диапазонов*

---

## 🚀 Основные возможности

- **Поиск по хешам** – загружайте список RIPEMD-160 (40 символов hex) из `.txt` или бинарного файла `.bin` (20 байт на хеш).
- **Гибкие диапазоны** – задавайте диапазон поиска в процентах от общего пространства ключей или напрямую в HEX (64 символа).
- **4 режима генерации ключей**:
  - *Секторный* – крайние воркеры идут от границ к центру, остальные случайно.
  - *Случайный* – все воркеры равномерно случайно перебирают весь диапазон.
  - *Случайный долями* – диапазон делится на части, каждый воркер случайно перебирает свою.
  - *Последовательный* – каждый воркер идёт последовательно со своим шагом; поддерживается сохранение состояния.
- **Карусель** – автоматический сдвиг диапазона на заданный шаг (в процентах или HEX) после проверки N миллионов ключей. Бесконечный циклический перебор или до 100%.
- **Многопоточность** – регулируемое количество потоков (до 512), динамическая статистика по каждому воркеру.
- **Визуализация** – реальное время: скорость проверки, найденные ключи, минимальный/максимальный проверенный ключ, загрузка памяти.
- **Сохранение результатов** – все найденные ключи и их адреса (Legacy, P2SH, Bech32) пишутся в `found_keys.txt` (или выбранный файл). Также сохраняется история сессий и конфигурация.

---

## 🖥️ Интерфейс (egui)

- Вкладка **«Настройки»** – выбор файла хешей, процентов/HEX диапазона, числа потоков, режима генерации, настройки карусели.
- Вкладка **«Карусель»** – лог событий сдвига диапазона (дата, старый/новый диапазон, время раунда, количество проверенных ключей).
- Вкладка **«Найденные ключи»** – таблица с приватными ключами, RIPEMD-160, Legacy и Bech32 адресами. Поддерживается копирование кликом.
- Вкладка **«Журнал сессий»** – история запусков: время, длительность, режим, диапазон, скорость, количество проверенных и найденных ключей.
- Вкладка **«Воркеры»** – для каждого потока отображаются последний проверенный ключ, а также min/max за последние 2 секунды.

## ⚙️ Технические детали

- **Язык:** Rust (безопасная работа с памятью, высокая производительность)
- **Криптография:** `secp256k1`, `ripemd`, `sha2`
- **GUI:** `eframe`/`egui` (легковесный, кроссплатформенный)
- **Декодирование адресов:** `base58`, `bech32`
- **Поддерживаемые форматы:** P2PKH (1...), P2SH (3...), Bech32 (bc1...), а также прямые P2PK (редкие, через версию 0x41/0x21)
- **Производительность:** до миллионов ключей в секунду в зависимости от числа потоков

## 📥 Установка и запуск

1. Установите Rust (если ещё нет): https://rustup.rs/
2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/ваш-аккаунт/bitcoin365-carousel-searcher.git
   cd bitcoin365-carousel-searcher
   ```
3. Соберите и запустите:
   ```bash
   cargo run --release
   ```
   *Первая компиляция может занять несколько минут.*

4. **Подготовьте файл с хешами** (один RIPEMD-160 на строку, 40 hex символов).  
   Пример:
   ```
   5d7e6c6b5c6a6f8b9a7b6d4c3e2f1a0b9c8d7e6f
   f3a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9
   ```

---

## 🧪 Пример использования

1. Загрузите файл с хешами (вкладка «Настройки» → «Файл хэшей»).
2. Выберите диапазон, например `0.000...%` до `0.000...%` (или укажите HEX).
3. Включите «Карусель», задайте лимит (например, 13 млн ключей) и шаг (например, `0.00000001%`).
4. Нажмите **«Старт»**.
5. Следите за прогрессом – при каждом достижении лимита диапазон автоматически сдвинется на шаг, и поиск продолжится.
6. Найденные ключи появятся на вкладке «Найденные ключи» и сохранятся в файл.

---

## 📁 Файлы конфигурации

- `app_config.json` – сохраняет последние настройки (хеш-файл, диапазон, число потоков и т.д.)
- `sessions.json` – история всех сессий (время, диапазон, результаты)
- `found_keys.txt` – все найденные ключи (формат: приватный_ключ \t RIPEMD-160 \t Legacy_uncompressed \t Legacy_compressed \t P2SH_uncompressed \t P2SH_compressed \t Bech32)

## ⚠️ Важно

- Инструмент предназначен для **образовательных целей** и **восстановления собственных утерянных ключей**.
- Не используйте для несанкционированного доступа к чужим средствам – это нарушает закон.
- Скорость поиска напрямую зависит от количества потоков и процессора. Для перебора больших диапазонов требуется время.
